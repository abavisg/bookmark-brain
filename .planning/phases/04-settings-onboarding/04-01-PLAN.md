    **Step 3 — Implement handlers** in `src/background/settings/settingsHandlers.ts`:

    `handleSaveSettings(payload)`:
    - If `payload.action === 'clear'`: call `chrome.storage.local.remove(['bbApiKeySecret'])`, then `chrome.storage.local.set({ bbHasApiKey: false, bbProvider: '' })`. Return `{ success: true }`.
    - Otherwise: save `bbProvider` and `bbOllamaBaseUrl` (default 'http://localhost:11434'). If `payload.apiKey` provided, save to `bbApiKeySecret` and set `bbHasApiKey: true`. If provider is 'ollama', set `bbHasApiKey: true` regardless of apiKey. Call `requeueFailedApiKeyJobs()` after save. Return `{ success: true }`.

    `handleGetSettings()`:
    - Read `bbProvider`, `bbHasApiKey`, `bbOllamaBaseUrl` from `chrome.storage.local`. NEVER read `bbApiKeySecret`.
    - Return `{ provider: bbProvider || '', hasApiKey: bbHasApiKey || false, ollamaBaseUrl: bbOllamaBaseUrl || 'http://localhost:11434' }`.

    `handleValidateApiKey(payload)`:
    - OpenAI: `fetch('https://api.openai.com/v1/models', { headers: { Authorization: 'Bearer ' + payload.apiKey } })`. Return `{ valid: true }` on 200, `{ valid: false, error: 'Invalid API key' }` on 401, `{ valid: false, error: response.statusText }` otherwise.
    - Anthropic: `fetch('https://api.anthropic.com/v1/messages', { method: 'POST', headers: { 'x-api-key': payload.apiKey, 'anthropic-version': '2023-06-01', 'content-type': 'application/json' }, body: JSON.stringify({ model: 'claude-sonnet-4-20250514', max_tokens: 1, messages: [{ role: 'user', content: 'hi' }] }) })`. Return `{ valid: true }` on 200, `{ valid: false, error: 'Invalid API key' }` on 401/403, `{ valid: false, error: response.statusText }` otherwise.
    - Ollama: `fetch((payload.ollamaBaseUrl || 'http://localhost:11434') + '/')`. Return `{ valid: true }` on 200, `{ valid: false, error: 'Ollama is not running at ' + url }` on error.
    - Wrap all fetches in try/catch; on network error return `{ valid: false, error: error.message }`.

    All three functions are async and exported. Use `chrome.storage.local.get` with promise wrappers (not callbacks) in the handlers. For test compatibility, the handlers should use `chrome.storage.local.get(keys)` which returns a Promise in MV3 service workers. In tests, mock implementations need to handle both callback and promise styles (setup.ts already stubs callback style; add promise-style returns in test mocks or use the callback pattern with a promise wrapper utility).
