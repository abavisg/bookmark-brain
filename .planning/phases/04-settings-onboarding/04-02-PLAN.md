    **Step 3 — SettingsPanel component.** Create `src/dashboard/components/SettingsPanel.tsx` with this structure:

    State:
    - `provider: 'openai' | 'anthropic' | 'ollama'` (from GET_SETTINGS on mount)
    - `apiKey: string` (input value, empty by default — NEVER populated from storage)
    - `ollamaBaseUrl: string` (default 'http://localhost:11434')
    - `hasApiKey: boolean` (from GET_SETTINGS)
    - `showKey: boolean` (password toggle, default false)
    - `validationStatus: 'idle' | 'validating' | 'valid' | 'invalid'`
    - `validationError: string`
    - `saving: boolean`

    On mount (`useEffect([], ...)`):
    - Call `sendMessage({ type: 'GET_SETTINGS' })` to load current provider/hasApiKey/ollamaBaseUrl
    - Set state from response: provider, hasApiKey, ollamaBaseUrl

    Layout (Tailwind, matching dashboard styling):
    - `<div className="max-w-lg">` container
    - `<h2>` heading: "Settings"
    - **Provider section:** Label "LLM Provider" + Select/dropdown with three options:
      - "OpenAI" (value: 'openai')
      - "Anthropic" (value: 'anthropic')
      - "Ollama (Local)" (value: 'ollama')
    - **API Key section** (shown when provider is 'openai' or 'anthropic'):
      - Label "API Key"
      - If `hasApiKey && apiKey === ''`: show status line with green checkmark: "API key saved" and a "Clear" button (text-red-600)
      - Password input with eye/eyeOff toggle button (Lucide `Eye`/`EyeOff` icons). Input placeholder: `sk-...` for OpenAI, `sk-ant-...` for Anthropic
      - Eye toggle button positioned `absolute right-2 top-1/2 -translate-y-1/2` inside a `relative` div
    - **Base URL section** (shown when provider is 'ollama'):
      - Label "Ollama Base URL"
      - Input with value `ollamaBaseUrl`, placeholder `http://localhost:11434`
    - **Action buttons row:**
      - "Save" button (indigo, primary) — disabled when: no provider selected, or (cloud provider and no apiKey and not hasApiKey)
      - "Validate" button (outline) — disabled when saving or no credentials entered
    - **Validation status:** Below buttons, conditionally rendered:
      - validating: spinner or "Validating..." text
      - valid: green text "API key verified" with checkmark
      - invalid: red text "Invalid key" + validationError

    Save handler:
    - Call `sendMessage({ type: 'SAVE_SETTINGS', payload: { provider, apiKey: apiKey || undefined, ollamaBaseUrl } })`
    - On success: set `hasApiKey: true`, clear `apiKey` input, set `saving: false`

    Validate handler:
    - Set `validationStatus: 'validating'`
    - Call `sendMessage({ type: 'VALIDATE_API_KEY', payload: { provider, apiKey: apiKey || undefined, ollamaBaseUrl } })`
    - On response: set `validationStatus` to 'valid' or 'invalid', set `validationError` from response.error

    Clear handler:
    - Call `sendMessage({ type: 'SAVE_SETTINGS', payload: { provider, action: 'clear' } })`
    - On success: set `hasApiKey: false`, clear all inputs, reset validation status
