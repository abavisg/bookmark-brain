  <action>
1. Create `public/icons/` directory.

2. Generate brain icon PNGs at 16, 32, 48, and 128px sizes. Approach options (try in order):
   - Option A: Use an SVG brain icon from tabler-icons (MIT licensed — `@tabler/icons` package or download the SVG directly). The icon is `IconBrain`. Convert SVG to PNG at each size using Sharp: `pnpm add -D sharp`, then write a small Node.js script that reads the SVG string, pipes through `sharp().resize(size).png().toFile()` for each size (16, 32, 48, 128). Delete the script after generation.
   - Option B: If tabler-icons is unavailable, create a simple brain SVG manually (a rounded shape with a center line and two curves suggesting brain hemispheres) and convert with Sharp.
   - Option C: If Sharp has issues, create simple colored placeholder PNGs (a solid circle with "BB" text or similar) using a canvas library or even just use a minimal valid PNG. The icon can be refined later — the important thing is that Chrome loads real PNG files at the correct sizes.

   IMPORTANT: Chrome manifest icons MUST be PNG format. SVG is NOT supported. Each file must be a valid PNG at its exact declared size.

3. Initialize shadcn/ui:
   - Run `pnpm dlx shadcn@latest init` — when prompted: use TypeScript, use default style, set base color, set CSS variables (yes), set `@/lib/utils` as the utils path, set `@/components/ui` as the components path
   - If the CLI hangs or fails interactively, create the config files manually:
     - `components.json` with correct paths for the project structure (aliases: `@/components`, `@/lib`, `@/hooks`)
     - `src/lib/utils.ts` with the `cn` helper function (clsx + tailwind-merge): `pnpm add clsx tailwind-merge`, then export `cn(...inputs: ClassValue[]) { return twMerge(clsx(inputs)) }`
   - Verify shadcn is working by running `pnpm dlx shadcn@latest add button` — this should create `src/components/ui/button.tsx`. If it works, keep the button (useful for testing). If it fails, troubleshoot the components.json config.
  </action>
