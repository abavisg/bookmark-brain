  <name>Task 1: Initialize project, install dependencies, configure build pipeline and tooling</name>
  <files>package.json, pnpm-lock.yaml, vite.config.ts, vitest.config.ts, manifest.json, tsconfig.json, tsconfig.app.json, tsconfig.node.json, biome.json, .gitignore, src/test/setup.ts</files>
  <action>
1. Initialize the project with pnpm:
   - Run `pnpm init` in the project root (or create package.json manually)
   - Set `"type": "module"` in package.json (required for Vite ESM imports)
   - Set `"name": "bookmark-brain"`, `"version": "0.1.0"`, `"private": true`

2. Install all dependencies:
   - Production: `pnpm add react react-dom tailwindcss`
   - Dev: `pnpm add -D vite @vitejs/plugin-react @crxjs/vite-plugin typescript @types/react @types/react-dom @types/chrome @tailwindcss/vite vitest @vitest/coverage-v8 @testing-library/react @testing-library/jest-dom jsdom vitest-chrome @biomejs/biome`
   - Note: Use `@biomejs/biome` (not just `biome`) — that is the correct npm package name

3. Create `manifest.json` at project root (MV3 manifest — single source of truth for crxjs):
   ```json
   {
     "manifest_version": 3,
     "name": "Bookmark Brain",
     "version": "0.1.0",
     "description": "AI-powered bookmark manager — find anything you've saved",
     "permissions": ["storage", "activeTab", "scripting", "alarms", "bookmarks"],
     "host_permissions": ["https://*/*", "http://*/*"],
     "action": {
       "default_popup": "src/popup/index.html",
       "default_title": "Bookmark Brain",
       "default_icon": {
         "16": "icons/icon16.png",
         "32": "icons/icon32.png",
         "48": "icons/icon48.png",
         "128": "icons/icon128.png"
       }
     },
     "options_page": "src/dashboard/index.html",
     "background": {
       "service_worker": "src/background/index.ts",
       "type": "module"
     },
     "content_scripts": [
       {
         "matches": ["https://*/*", "http://*/*"],
         "js": ["src/content/index.ts"],
         "run_at": "document_idle"
       }
     ],
     "icons": {
       "16": "icons/icon16.png",
       "32": "icons/icon32.png",
       "48": "icons/icon48.png",
       "128": "icons/icon128.png"
     }
   }
   ```

4. Create `vite.config.ts` — Vite + crxjs + react + tailwindcss plugins. Use `@` path alias resolving to `src/`. Do NOT add `rollupOptions.input` — crxjs reads entries from manifest.json. Include deterministic filenames in `rollupOptions.output` (`entryFileNames: '[name].js'`, etc.). Follow the exact pattern from RESEARCH.md.

5. Create `vitest.config.ts` — SEPARATE from vite.config.ts (crxjs is incompatible with Vitest). Include: `@vitejs/plugin-react`, `environment: 'jsdom'`, `globals: true`, `setupFiles: ['./src/test/setup.ts']`, `css: true`, `@` path alias. Follow the exact pattern from RESEARCH.md.

6. Create `src/test/setup.ts` — Chrome API mocks:
   ```typescript
   import * as chrome from 'vitest-chrome'
   Object.assign(global, chrome)
   ```

7. Create TypeScript configs — THREE files following the split tsconfig pattern from RESEARCH.md:
   - `tsconfig.json`: Root with `"files": []` and `"references"` to app and node configs
   - `tsconfig.app.json`: DOM context — `lib: ["ES2022", "DOM", "DOM.Iterable"]`, `jsx: "react-jsx"`, includes `src/popup/**/*`, `src/dashboard/**/*`, `src/content/**/*`, `src/shared/**/*`. Add `types: ["chrome"]` and `paths: {"@/*": ["src/*"]}`
   - `tsconfig.node.json`: WebWorker context — `lib: ["ES2022", "WebWorker"]`, `composite: true`, includes `src/background/**/*`, `src/shared/**/*`, `vite.config.ts`, `manifest.json`. Add `types: ["chrome"]` and `paths: {"@/*": ["src/*"]}`
   - IMPORTANT: Both app and node configs need `"resolveJsonModule": true` for manifest.json import in vite.config.ts

8. Create `biome.json` — Follow the exact config from RESEARCH.md (recommended rules, space indent 2, single quotes, no semicolons, ignore dist/ and node_modules/).

9. Add scripts to package.json:
   ```json
   {
     "dev": "vite",
     "build": "tsc -b && vite build",
     "lint": "biome check src/",
     "format": "biome format --write src/",
     "test": "vitest",
     "test:run": "vitest run",
     "test:coverage": "vitest run --coverage",
     "type-check": "tsc -b --noEmit"
   }
   ```

10. Create `.gitignore` with: `node_modules/`, `dist/`, `coverage/`, `.vite/`, `*.local`

<task type="auto">
  <name>Task 2: Create all extension entry points, typed message bus, and placeholder UIs</name>
  <files>src/background/index.ts, src/content/index.ts, src/popup/index.html, src/popup/index.tsx, src/popup/index.css, src/popup/App.tsx, src/dashboard/index.html, src/dashboard/index.tsx, src/dashboard/index.css, src/dashboard/App.tsx, src/shared/types/messages.ts, src/shared/messages/bus.ts</files>
  <action>
1. Create `src/shared/types/messages.ts` — The typed message bus contract. This is the most important file in the shared layer. Follow the discriminated union pattern from RESEARCH.md exactly:
   - Define `AppMessage` as a discriminated union with `type` field. Include at minimum: `PING`, `GET_STATUS`
   - Define `AppResponse<T extends AppMessage>` as a conditional type mapping each message type to its response shape
   - Export both types (other plans depend on extending AppMessage)

2. Create `src/shared/messages/bus.ts` — The typed sendMessage wrapper:
   - Export `sendMessage<T extends AppMessage>(message: T): Promise<AppResponse<T>>`
   - Wraps `chrome.runtime.sendMessage` with proper error handling via `chrome.runtime.lastError`
   - Import types from `@/shared/types/messages`

3. Create `src/background/index.ts` — Service worker entry:
   - Import `AppMessage` from `@/shared/types/messages`
   - Add `chrome.runtime.onMessage.addListener` with a switch on `message.type`
   - Handle `PING` (respond `{ alive: true }`), `GET_STATUS` (respond with manifest version)
   - Add `chrome.runtime.onInstalled.addListener` that creates a keepalive alarm: `chrome.alarms.create('keepalive', { periodInMinutes: 0.4 })`
   - Add `chrome.alarms.onAlarm.addListener` with a stub for future queue processing
   - Return `false` from the listener for synchronous responses (return `true` only for async)

4. Create `src/content/index.ts` — Content script entry (minimal placeholder):
   - Add a single console.log: `console.log('[Bookmark Brain] Content script loaded')`
   - This file will be expanded in Phase 3 (bookmark saving)

5. Create popup entry files:
   - `src/popup/index.html`: Standard HTML5 boilerplate with `<div id="root">` and `<script type="module" src="./index.tsx">`. Title: "Bookmark Brain"
   - `src/popup/index.css`: Contains `@import "tailwindcss";` (Tailwind 4 CSS-only entry)
   - `src/popup/index.tsx`: React 18 createRoot mounting App in StrictMode, imports `./index.css`
   - `src/popup/App.tsx`: Simple placeholder component — a div with "Bookmark Brain" heading and "Bookmark Brain is ready" text. Use basic Tailwind classes. Width 380px, min-height 200px. Include `dark:` variants for background and text colors (system dark mode — no config needed with Tailwind 4).

6. Create dashboard entry files:
   - `src/dashboard/index.html`: Same pattern as popup HTML
   - `src/dashboard/index.css`: Contains `@import "tailwindcss";`
   - `src/dashboard/index.tsx`: React 18 createRoot mounting App in StrictMode, imports `./index.css`
   - `src/dashboard/App.tsx`: Layout skeleton with header, sidebar, and main content area. Header: "Bookmark Brain" title. Sidebar: placeholder nav items (Library, Search, Settings — just text, no routing). Main: empty content area with "Welcome to Bookmark Brain" placeholder. Use Tailwind grid/flex layout. Include `dark:` variants.
